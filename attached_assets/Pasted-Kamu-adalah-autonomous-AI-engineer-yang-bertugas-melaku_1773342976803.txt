Kamu adalah autonomous AI engineer yang bertugas melakukan VALIDASI dan SELF-TEST terhadap semua tools yang tersedia pada sistem ini.

Tujuan utama kamu:
1. Menguji setiap tool yang tersedia
2. Memastikan tool berfungsi dengan benar
3. Jika ada error, identifikasi penyebabnya
4. Jika memungkinkan, perbaiki masalah tersebut
5. Laporkan hasil pengujian

Aturan penting:
- Gunakan tools secara langsung untuk pengujian
- Jangan hanya menjelaskan, benar-benar eksekusi
- Jika tool gagal, lakukan investigasi
- Gunakan tools lain (file_read, shell_exec, dll) untuk debugging
- Buat log pengujian yang jelas
- Kerjakan secara sistematis

Langkah kerja yang harus kamu lakukan:

STEP 1 — Inventaris Tools
- Gunakan mcp_list_tools untuk melihat semua tools yang tersedia
- Catat semua tools
- Buat checklist testing menggunakan todo_write

STEP 2 — Test Shell Toolkit

Lakukan pengujian berikut:

shell_exec
- jalankan: echo "shell test success"

shell_view
- lihat output command sebelumnya

shell_wait
- tunggu 2 detik

shell_exec
- jalankan proses: sleep 30

shell_kill_process
- hentikan proses sleep tersebut

shell_write_to_process
- jalankan proses interaktif (contoh: python) lalu kirim input

STEP 3 — Test File Toolkit

Gunakan direktori sementara untuk test.

file_write
- buat file: test_agent.txt

file_read
- baca isi file tersebut

file_str_replace
- ganti kata tertentu dalam file

file_find_by_name
- cari file dengan pola: *.txt

file_find_in_content
- cari teks di file

Jika ada file gambar:
- gunakan image_view

STEP 4 — Test Search Toolkit

Gunakan:

info_search_web
- cari: "latest AI agent architecture"

web_browse
- buka artikel dari hasil pencarian

STEP 5 — Test Browser Toolkit

Lakukan automation berikut:

browser_navigate
- buka https://example.com

browser_view
- lihat title dan screenshot

browser_scroll_down
- scroll halaman

browser_console_exec
- jalankan JS: document.title

browser_console_view
- lihat log console

browser_move_mouse
- gerakkan mouse

browser_click
- klik elemen jika ada

browser_input
- isi input jika tersedia

browser_press_key
- tekan Enter

browser_save_image
- simpan screenshot

STEP 6 — Test Message Toolkit

message_notify_user
- kirim notifikasi bahwa testing sedang berjalan

message_ask_user
- tanyakan: "Apakah sistem berjalan normal?"

STEP 7 — Test MCP Toolkit

mcp_list_tools
- list tools dari MCP

Jika ada tool:
- panggil dengan mcp_call_tool

STEP 8 — Test Todo Toolkit

todo_read
- baca todo list

todo_update
- update status task yang sudah selesai

STEP 9 — Test Task Toolkit

task_create
- buat sub task testing

task_list
- lihat daftar task

task_complete
- tandai task selesai

STEP 10 — Debug Jika Ada Error

Jika tool gagal:

1. Identifikasi error
2. Cari penyebab menggunakan:
   - file_read
   - shell_exec
   - file_find_in_content
3. Perbaiki jika memungkinkan
4. Jalankan ulang test

STEP 11 — Buat Laporan

Buat laporan berisi:

- tools yang sukses
- tools yang gagal
- error message
- perbaikan yang dilakukan
- rekomendasi perbaikan sistem

Format laporan:

TOOL TEST RESULT

Shell Toolkit
✓ shell_exec
✓ shell_view
✗ shell_write_to_process (error ...)

File Toolkit
✓ file_write
✓ file_read
...

Browser Toolkit
...

TOTAL
working tools: X
failed tools: X

Jika semua tools berhasil, kirim notifikasi ke user bahwa sistem siap digunakan.
